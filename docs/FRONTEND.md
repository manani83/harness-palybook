# 프런트엔드 기준

## 목적
이 가이드는 사용자에게 보이는 인터페이스, 콘텐츠 표면, 상호작용 워크플로에 변경이 닿을 때 사용한다.

## 프런트엔드 기준
- 읽기 쉬운 계층 구조와 분명한 조작 가능성을 우선한다.
- 레이아웃은 처음부터 반응형으로 만든다.
- 빈 상태, 로딩, 성공, 오류 상태를 명시적으로 제공한다.
- 의미론적 HTML과 접근 가능한 상호작용 패턴을 사용한다.
- 문구는 간결하고 사실적이며 행동 지향적으로 쓴다.
- 핵심 작업을 가리는 시각적 잡음을 피한다.
- 간격, 타이포그래피, 색상 토큰을 일관되게 사용한다.
- 파괴적이거나 되돌릴 수 없는 작업은 명확하게 드러낸다.
- 예측 가능한 탐색을 예상 밖의 상호작용보다 우선한다.
- 핵심 경로를 측정하고 빠르게 유지한다.

## 검토 체크리스트
- 키보드와 스크린 리더 접근이 유지되는가?
- 주요 행동이 분명한가?
- 상태 전환이 결정적인가?
- 오류 메시지가 복구 방법을 설명하는가?
- 작은 폭과 큰 폭 모두에서 내용이 읽히는가?
