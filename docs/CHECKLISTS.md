# 체크리스트

## 목적
하네스 업무 흐름에서 빠뜨리면 안 되는 확인 항목을 단계별로 고정한다.

## 사용 원칙
- 해당 단계의 체크리스트를 먼저 보고 문서를 쓴다.
- 체크되지 않은 항목은 기본적으로 미완료로 본다.
- 보안과 신뢰성은 선택 항목이 아니라 기본 항목이다.
- 설명이 길어지면 체크리스트보다 문서를 보완한다.

## 1. 인테이크 체크리스트
- 문제를 한 문장으로 말할 수 있는가?
- 영향을 받는 사용자나 운영자가 분명한가?
- 지금 해야 하는 이유가 있는가?
- 이 요청이 새 명세, 기존 명세 수정, 또는 기술 부채 중 어디에 속하는가?
- 관련 문서가 이미 있는가?

## 2. 제품 명세 체크리스트
- 문제와 원하는 결과가 분명한가?
- 사용자와 사용 맥락이 적혀 있는가?
- 목표와 비목표가 구분되어 있는가?
- 성공 지표가 측정 가능하게 적혀 있는가?
- 대안과 엣지 케이스를 검토했는가?
- 열린 질문이 빠짐없이 적혀 있는가?

## 3. 설계 검토 체크리스트
- 추천하는 접근법이 분명한가?
- 왜 그 방법을 고르는지 설명했는가?
- 트레이드오프와 위험이 명시됐는가?
- 마이그레이션 또는 롤아웃 경로가 있는가?
- 롤백이나 대체 경로가 있는가?
- 검증 방법과 기대 신호가 적혀 있는가?
- 보안과 신뢰성 영향을 따로 보았는가?

## 4. 실행 계획 체크리스트
- 목표, 범위, 완료 기준이 명확한가?
- 마일스톤이 실제 작업 단위로 쪼개져 있는가?
- 담당자가 분명한가?
- 의존성과 차단 요소가 적혀 있는가?
- 각 단계의 검증 방법이 있는가?
- 위험과 롤백이 적혀 있는가?
- 관련 제품, 설계, 참조 문서가 링크돼 있는가?

## 5. 진행 중 체크리스트
- 결정과 범위 변경이 즉시 기록되는가?
- 새로 드러난 위험이나 가정이 반영되는가?
- 차단 요소가 명확히 보이는가?
- 생성 문서와 참조 문서가 분리돼 있는가?
- 기술 부채가 별도로 추적되는가?

## 6. 완료 체크리스트
- 결과가 원래 목표와 비교돼 설명됐는가?
- 무엇을 검증했는지 근거가 남았는가?
- 남은 일이나 부채가 정리됐는가?
- 완료 계획이 `docs/exec-plans/completed/`로 옮겨졌는가?
- 인덱스와 교차 링크가 최신 상태인가?
- 용어가 용어집과 일치하는가?

## 7. 문서 유지보수 체크리스트
- 첫 문단이 목적을 말하는가?
- 읽는 사람이 다음 행동을 알 수 있는가?
- 중복되거나 충돌하는 지침이 없는가?
- 생성된 콘텐츠가 생성물로 표시돼 있는가?
- 새 문서를 추가했으면 인덱스도 갱신했는가?
