# 제품 명세

제품 명세는 설계나 실행을 시작하기 전에 문제와 원하는 결과를 설명한다.

## 이 섹션의 용도
- 새로운 워크플로
- 동작 변경
- 사용자에게 보이는 개선
- 우선순위 결정

## 권장 구조
- 문제
- 페르소나
- 목표와 비목표
- 제안 방안
- 성공 지표
- 검토한 대안
- 열린 질문

- [셀러용 가격 모니터링 + 경쟁사 추적 서비스](./seller-price-monitoring.md)

## 새 명세를 추가할 때
- 파일 이름은 내부 구현이 아니라 사용자 문제를 기준으로 짓는다
- 결과 중심으로 쓴다
- 대응되는 설계 문서나 계획 문서가 있으면 링크한다
