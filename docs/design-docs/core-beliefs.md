# 핵심 신념

하네스 엔지니어링은 문서화된 체계가 실제 체계를 더 쉽게 구축하고, 검토하고, 운영할 수 있게 할 때 가장 잘 작동한다.

## 신념
1. 문서화된 의도는 일급 산출물이다.
2. 가장 작은 유용한 워크플로가 크고 정의되지 않은 시스템보다 낫다.
3. 검증할 수 없는 변경은 준비되지 않은 것이다.
4. 운영상의 진실은 의도된 동작보다 더 중요하다.
5. 재사용은 명확성을 높일 때만 가치가 있다.
6. 오래 가는 시스템에는 명시적인 소유권과 복구 경로가 필요하다.
7. 가정은 보이게 만들어야 한다.
8. 내구성 있는 인터페이스와 버전이 있는 계약을 선호한다.
9. 위험을 줄이는 가장 빠른 방법은 범위를 좁히는 것이다.
10. 좋은 문서는 조정 비용을 줄인다.

## 실무에서의 의미
- 결정이 민간전승이 되기 전에 기록한다.
- 제품, 설계, 실행, 운영을 서로 연결해 둔다.
- 생성된 사실과 사람의 판단을 서로 다른 종류의 진실로 다룬다.
- 현재 대화가 아니라 다음 독자를 위해 쓴다.
- 검토 가능하게 만드는 최소한의 구조를 사용한다.
