# 셀러용 가격 모니터링 화면 와이어프레임

이 문서는 셀러용 가격 모니터링 서비스의 핵심 화면을 텍스트 기반 와이어프레임으로 정리한다. 실제 비주얼 디자인이 아니라 정보 구조, 시선 흐름, 조작 우선순위를 고정하는 것이 목적이다.

## 원칙
- 요약을 먼저 보여주고, 그다음 대응할 수 있게 만든다.
- 데이터 신선도와 수집 시각을 항상 노출한다.
- 데스크톱은 3단 구조, 모바일은 1열 누적 구조로 설계한다.
- 빈 상태, 로딩, 오류, 권한 없음 상태를 숨기지 않는다.
- 핵심 행동은 `메모`, `무시`, `핀 고정`, `알림 조정`처럼 짧고 분명하게 만든다.

## 공통 레이아웃
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Logo | Workspace ▼ | Date Range ▼ | Search ▢ | Alerts | User Menu           │
├──────────────────────────────────────────────────────────────────────────────┤
│ Page Title / Subcopy                                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│ Main Content                                                                  │
└──────────────────────────────────────────────────────────────────────────────┘
```

공통적으로 상단에는 워크스페이스 전환, 기간 선택, 검색, 알림 상태, 사용자 메뉴를 둔다. 모든 표와 카드에는 마지막 수집 시각을 표시한다.

## 1. 랜딩페이지
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Nav: 제품 소개 | 기능 | 요금제 | 로그인 | 무료 체험                          │
├──────────────────────────────────────────────────────────────────────────────┤
│ Hero                                                                         │
│ 경쟁사 가격 변동을 놓치지 마세요                                             │
│ 쿠팡·스마트스토어 가격, 쿠폰, 품절, 리뷰 변화를 한 번에 추적                │
│ [무료 체험 시작]  [데모 보기]                                                │
├──────────────────────────────────────────────────────────────────────────────┤
│ Problem Strip | Value Cards | Demo Screenshot | Pricing | FAQ                │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 히어로는 문제, 해결 방식, CTA를 한 화면에 묶는다.
- 데모 스크린샷은 대시보드 요약과 상세 화면을 함께 보여준다.
- 요금제는 Starter, Growth, Agency를 나란히 비교한다.

## 2. 로그인 / 가입
```text
┌──────────────────────────────────────────────┐
│ 왼쪽: 제품 요약 / 신뢰 문구                  │
│ 오른쪽: 로그인 폼                            │
│ - 이메일                                    │
│ - 비밀번호                                  │
│ - 로그인 버튼                               │
│ - 회원가입 링크                             │
└──────────────────────────────────────────────┘
```

- 가입 후 첫 진입은 워크스페이스 생성 또는 초대 수락 흐름으로 연결한다.
- 비밀번호 기반 로그인 외에 추후 매직 링크를 붙일 수 있도록 문구를 중립적으로 유지한다.

## 3. 대시보드
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Workspace | Range | Sync status | Export | User Menu                          │
├──────────────────────────────────────────────────────────────────────────────┤
│ [가격 하락 경쟁사 12] [내 상품보다 싼 경쟁사 8] [마진 위험 3] [품절 2] [리뷰 5] │
├──────────────────────────────────────────────────────────────────────────────┤
│ [가격 추이 차트 / 최저가 이동 / 이벤트 밀도]   │ [오늘 바로 대응할 상품]      │
│                                               │ - 상품명 / 위험 / 액션      │
│                                               │ - 상품명 / 위험 / 액션      │
├──────────────────────────────────────────────────────────────────────────────┤
│ [최근 이벤트 타임라인]                       │ [요약 메일 미리보기]         │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 최상단 카드 5개는 사용자가 오늘 해야 할 행동의 우선순위를 알려준다.
- 오른쪽 리스트는 `가격 하락`, `마진 위험`, `품절 기회`처럼 즉시 조치가 필요한 항목만 노출한다.
- 차트는 기간별 가격 변화를 보여주고, 카드 클릭 시 해당 상품 상세로 이동한다.

## 4. 상품 상세 모니터링
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 내 상품명 / 채널 / 현재가 / 최소 마진가 / 시장 최저가 차이 / [메모] [핀]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [경쟁상품 테이블]                                                            │
│ 채널 | 경쟁사 | 상품명 | 현재가 | 배송비 | 쿠폰반영가 | 리뷰 | 평점 | 품절 | 수집시각 │
├──────────────────────────────────────────────────────────────────────────────┤
│ [가격 이력 그래프]        │ [이벤트 이력]        │ [알림 이력 / 메모]        │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 테이블은 실질 판매가 기준 정렬을 기본으로 한다.
- 이벤트 이력은 가격 변화, 품절, 재입고, 리뷰 급증, 평점 변화를 한 줄 카드로 보여준다.
- 알림 이력에는 발송 채널, 발송 시각, 읽음 상태를 함께 표시한다.

## 5. 이벤트 센터
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Filters: 가격 하락 | 가격 상승 | 품절 | 재입고 | 리뷰 증가 | 평점 하락       │
├──────────────────────────────────────────────────────────────────────────────┤
│ [이벤트 카드]                                                                │
│ 무엇이 바뀌었는가                                                            │
│ 언제 바뀌었는가                                                              │
│ 얼마나 바뀌었는가                                                            │
│ [메모] [무시] [핀 고정]                                                      │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 이벤트 목록은 감지 순서가 아니라 중요도 순으로 정렬한다.
- `무시`는 상태를 숨기는 것이 아니라 후속 필터링 기준으로 남긴다.
- `핀 고정`은 회의나 보고서에서 다시 볼 항목을 표시한다.

## 6. 알림 설정
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Rule Cards                                                                   │
│ - 경쟁사 가격이 3% 이상 내려가면 알림                                       │
│ - 내 최소 마진보다 낮아지면 즉시 알림                                       │
│ - 품절 전환 시 즉시 알림                                                    │
│ - 하루 1회 오전 8시 요약 메일                                                │
├──────────────────────────────────────────────────────────────────────────────┤
│ Channels: Email | Slack | KakaoWork | Webhook                                │
│ Quiet hours | Category filters | Per-product overrides                        │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 알림 규칙은 상품별 오버라이드와 조직 공통 규칙으로 나눈다.
- 요약 메일과 즉시 알림은 역할을 분리한다.

## 7. 워크스페이스 / 멤버 / 결제
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Workspace Settings                                                            │
├──────────────────────────────────────────────────────────────────────────────┤
│ Members Table: 이름 | 이메일 | 역할 | 상태 | 초대일 | 액션                    │
│ Invite Form: 이메일 ▢ | 역할 ▼ | [초대 보내기]                                │
├──────────────────────────────────────────────────────────────────────────────┤
│ Plan Card: 현재 플랜 / 사용량 / 초과 사용 / 업그레이드 CTA                    │
│ Billing: 결제 수단 / 인보이스 / 청구서 다운로드                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 멤버 관리와 요금제 관리는 같은 설정 화면에 묶는다.
- 초대 대기 상태와 활성 상태를 분리해서 보여준다.

## 모바일 대응
- 상단 네비게이션은 햄버거 메뉴 또는 하단 탭으로 축약한다.
- 요약 카드 5개는 가로 스크롤 또는 2열 스택으로 배치한다.
- 상세 화면의 경쟁상품 테이블은 카드형 리스트로 바뀐다.
- 이벤트 센터는 필터 칩 + 세로 타임라인 구조로 바뀐다.

## 공통 상태
- 빈 상태: 아직 등록된 상품이 없을 때는 샘플 CTA와 입력 가이드를 보여준다.
- 로딩 상태: 수집 중인 시간과 마지막 성공 시각을 함께 보여준다.
- 오류 상태: 수집 실패, 권한 없음, 결제 만료를 분리해서 보여준다.
- 오프라인 상태: 최근 캐시와 마지막 동기화 시각을 유지한다.

## 화면 우선순위
1. 대시보드
2. 상품 상세 모니터링
3. 이벤트 센터
4. 알림 설정
5. 워크스페이스 / 멤버 / 결제
6. 랜딩페이지
7. 로그인 / 가입
