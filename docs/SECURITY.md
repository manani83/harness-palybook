# 보안

## 목적
데이터, 자격 증명, 접근, 신뢰 경계에 닿는 작업에는 이 가이드를 사용한다.

## 보안 원칙
- 비밀 값은 최소화하고 문서와 로그에서 제외한다.
- 필요한 최소 권한만 부여한다.
- 외부 입력은 신뢰하지 않는다.
- 권한 부여는 암시하지 말고 명시한다.
- 안전한 기본값을 선호한다.
- 의존성과 공급망 위험을 추적한다.
- 기본값으로 개인정보를 보호한다.

## 위협 모델 질문
- 자산은 무엇인가?
- 공격자 또는 실수로 오용할 주체는 누구인가?
- 신뢰 경계는 어디인가?
- 어떤 데이터가 민감한가?
- 유출, 변조, 악용의 영향은 무엇인가?

## 기본 체크리스트
- 비밀 값은 저장소 밖에 보관된다
- 민감한 값은 마크다운에 커밋하지 않는다
- 접근 경로가 문서화돼 있다
- 인증과 권한 부여가 설명돼 있다
- 의존성 업데이트를 검토한다
- 로그에 토큰, 키, 개인 사용자 데이터가 남지 않는다
- 보관 및 삭제 기대치가 명확하다

## 검토 요구사항
- 새 데이터를 노출하는 변경이라면 그 노출을 명시적으로 문서화한다.
- 권한을 바꾸는 변경이라면 최소 권한 동작을 검증한다.
- 서드파티 서비스에 의존한다면 신뢰와 실패 영향을 적는다.
