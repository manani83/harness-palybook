# 이름 규칙

## 목적
이 문서는 기획팀 사람별 폴더와 관련 문서의 이름을 일관되게 유지하기 위한 규칙을 정의한다.

## 규칙
- 사람 폴더는 `docs/planning/people/<name-slug>/` 형식을 사용한다.
- 폴더 슬러그는 소문자, 숫자, 하이픈만 사용한다.
- Jira 카드 파일은 `YYYY-MM-DD_<jira-key>_<short-slug>.md` 형식을 사용한다.
- 날짜는 카드가 처음 등록된 날짜를 쓴다.
- `jira-key`는 대문자 프로젝트 키와 번호를 그대로 쓴다.
- `short-slug`는 내용을 짧게 줄인 소문자 하이픈 표기다.
- 상태가 바뀌어도 파일명은 바꾸지 않는다.
- 정책 파일은 kebab-case를 사용한다.
- 임시 파일이나 개인 메모도 의미가 드러나는 이름을 쓴다.

## 권장 방식
- 사람 이름은 팀 내에서 통용되는 표기 대신 파일 시스템에 안정적인 슬러그를 쓴다.
- Jira 키는 원문 그대로 유지해 추적성을 높인다.
- 이름을 바꿔야 한다면 새 파일을 만들지 말고 기존 파일을 이동하거나 갱신한다.

## 예시
- `2026-04-15_SPAG-9999_planning-structure-sample.md`
- `2026-04-15_SPAG-6823_bos-storage-normalization.md`
