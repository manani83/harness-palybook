# Jira 생명주기

## 목적
이 문서는 기획팀이 Jira 항목을 `active`, `blocked`, `done`으로 관리할 때의 이동 규칙을 정의한다.

## 상태
| 상태 | 의미 | 이동 조건 |
| --- | --- | --- |
| `active` | 지금 처리 중인 항목 | 담당자가 작업을 시작했을 때 |
| `blocked` | 외부 의존성이나 결정이 막고 있는 항목 | 필요한 입력이나 승인이 없을 때 |
| `done` | 처리와 인수인계가 끝난 항목 | 결과 링크와 마무리 메모가 남았을 때 |

## 규칙
- 한 Jira 항목은 한 시점에 하나의 상태만 가진다.
- 상태가 바뀌면 파일을 복제하지 말고 해당 상태 폴더로 이동한다.
- 완료된 항목은 결과 링크, 최종 판단, 남은 후속 작업을 함께 남긴다.
- 실행 계획이 필요하면 Jira 파일 안에만 적지 말고 `docs/exec-plans/`로 넘긴다.
