# 셀러용 가격 모니터링 QA 체크리스트

이 문서는 셀러용 가격 모니터링 + 경쟁사 추적 서비스의 핵심 사용자 시나리오를 수동 스모크 테스트할 때 쓰는 체크리스트다. 기능 구현 문서가 아니라, 문서와 구현이 맞물려 있는지 확인하는 검증 기준을 고정한다.

## 범위
- 로그인과 워크스페이스 진입
- 대시보드 요약 카드
- 모니터링 상품 등록과 상세 조회
- 경쟁상품 등록과 스냅샷 조회
- 이벤트 목록과 이벤트 상태 변경
- 알림 규칙과 알림 상태
- 리포트 조회

## 사전 조건
- 테스트용 조직 1개가 있다.
- 테스트용 사용자 1명이 조직에 속해 있다.
- 테스트용 모니터링 상품 1개와 경쟁상품 3개가 있다.
- 첫 스냅샷이 저장돼 있다.
- 알림 채널 1개가 연결돼 있다.

## 공통 통과 기준
- 응답이 성공이면 화면에 데이터가 바로 보인다.
- 응답이 실패하면 오류 메시지와 원인이 눈에 보인다.
- 상태 변경은 새로고침 후에도 유지된다.
- 권한이 없는 사용자는 접근할 수 없다.
- 수집 실패나 알림 실패는 숨겨지지 않는다.

## 체크리스트
### 1. 인증과 워크스페이스
- [ ] 로그인 성공 시 대시보드로 이동한다.
- [ ] 잘못된 비밀번호는 `UNAUTHORIZED` 오류로 막힌다.
- [ ] 토큰 만료 후 새로고침 시 재인증 흐름이 동작한다.
- [ ] 조직 목록이 현재 사용자 권한에 맞게 보인다.
- [ ] 초대 수락 후 멤버 상태가 `active`로 바뀐다.

### 2. 대시보드
- [ ] 오늘 요약 카드가 수치와 함께 보인다.
- [ ] 가격 하락, 최저가 경쟁, 마진 위험, 품절, 리뷰 급증이 구분된다.
- [ ] 오늘 바로 대응할 상품 리스트가 나온다.
- [ ] 요약 카드 클릭이 상세 또는 이벤트 화면으로 연결된다.

### 3. 모니터링 상품
- [ ] 모니터링 상품을 새로 등록할 수 있다.
- [ ] 경쟁상품 링크를 3개 이상 등록할 수 있다.
- [ ] 상세 화면에서 내 상품 정보와 최소 마진 가격이 보인다.
- [ ] 가격 이력 그래프와 마지막 수집 시각이 보인다.
- [ ] 메모 수정과 핀 고정이 동작한다.

### 4. 경쟁상품과 스냅샷
- [ ] 경쟁상품 목록이 채널별로 구분된다.
- [ ] 현재가, 배송비, 쿠폰 반영가가 함께 보인다.
- [ ] 품절과 재입고가 상태로 표현된다.
- [ ] 리뷰 수와 평점 변화가 시간축으로 보인다.
- [ ] 최신 스냅샷이 없으면 빈 상태가 나온다.

### 5. 이벤트
- [ ] 가격 하락 이벤트가 생성된다.
- [ ] 가격 상승 이벤트가 생성된다.
- [ ] 품절 전환과 재입고 이벤트가 생성된다.
- [ ] 리뷰 급증 이벤트가 생성된다.
- [ ] 이벤트를 읽음 처리할 수 있다.
- [ ] 이벤트에 메모를 남길 수 있다.
- [ ] 이벤트를 핀 고정할 수 있다.

### 6. 알림
- [ ] 가격 하락 임계치에 맞는 알림 규칙을 저장할 수 있다.
- [ ] 최소 마진 이하일 때 즉시 알림이 만들어진다.
- [ ] 알림 상태가 `queued`, `sent`, `delivered`, `failed` 중 하나로 보인다.
- [ ] 실패한 알림에는 실패 사유가 남는다.
- [ ] 하루 요약 리포트가 지정된 채널로 전송된다.

### 7. 리포트
- [ ] 일간 리포트가 조회된다.
- [ ] 리포트에 당일 변동 수와 주요 이벤트가 요약된다.
- [ ] 보고서 다운로드가 권한에 따라 제한된다.

## 중단 기준
- 인증이나 조직 권한이 꼬이면 나머지 시나리오를 진행하지 않는다.
- 스냅샷이 없으면 이벤트와 리포트 검증을 중단한다.
- 알림 전송 실패가 연속되면 원인 파악 전까지 기능 통과로 보지 않는다.

## 증거
- 테스트 계정과 조직 ID
- 성공/실패 화면 캡처
- 이벤트 생성 시간
- 알림 전송 로그
- 리포트 응답 예시
