# 실행 계획

## 목적
이 문서는 의도에서 완료까지 이어질 만큼 구체적인 실행 계획을 작성할 때 사용한다.

## 계획을 작성할 때
- 여러 단계가 필요할 때
- 의미 있는 위험이나 의존성이 있을 때
- 변경에 검증이나 롤아웃이 필요할 때
- 작업이 짧은 편집 세션보다 오래 이어질 때

## 계획 템플릿
- 제목
- 목표
- 맥락
- 범위
- 비목표
- 제안 접근법
- 마일스톤
- 검증
- 위험
- 롤백 또는 대체 경로
- 완료 기준

## 진행 중인 계획 규칙
- 작업이 진행되는 동안 계획을 최신 상태로 유지한다.
- 결정이 생길 때마다 기록한다.
- 범위가 바뀌면 상태도 갱신한다.
- 관련 제품, 설계, 참조 문서와 링크를 건다.

## 완료된 계획 규칙
- 무엇이 바뀌었는지 요약한다.
- 무엇을 검증했는지 기록한다.
- 남은 후속 작업이나 부채를 적는다.
- 끝난 계획은 `docs/exec-plans/completed/`로 옮긴다.
